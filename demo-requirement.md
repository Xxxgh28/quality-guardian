# 用户注册登录系统需求文档

## 1. 简介
本系统旨在为用户提供安全、便捷的注册和登录功能，支持传统的账号密码方式以及第三方 OAuth 登录（GitHub, Google）。

## 2. 功能需求

### 2.1 用户注册
- **输入项**：用户名、邮箱、密码、确认密码。
- **校验规则**：
  - 用户名：4-20个字符，仅允许字母、数字、下划线。
  - 邮箱：必须符合标准邮箱格式，且系统内唯一。
  - 密码：至少8位，包含大小写字母和数字。
  - 确认密码：必须与密码一致。
- **流程**：
  1. 用户填写注册表单并提交。
  2. 后端验证输入数据的合法性。
  3. 验证通过后，发送验证邮件到用户邮箱。
  4. 用户点击邮件中的链接激活账号。
- **异常处理**：
  - 邮箱已被注册：提示“该邮箱已被使用”。
  - 格式错误：实时提示具体的格式错误信息。

### 2.2 用户登录
- **方式**：账号密码登录、OAuth 登录。
- **账号密码登录**：
  - 输入：邮箱/用户名、密码。
  - 包含“记住我”选项，保持登录状态 7 天。
  - 连续输错密码 5 次，锁定账号 15 分钟。
- **OAuth 登录**：
  - 支持 GitHub 和 Google 账号授权。
  - 首次 OAuth 登录自动创建关联账号。

### 2.3 密码找回
- **流程**：
  1. 用户点击“忘记密码”。
  2. 输入注册邮箱。
  3. 系统发送重置密码链接（有效期 30 分钟）。
  4. 用户点击链接，跳转到重置密码页面。
  5. 输入新密码并确认。
- **安全要求**：
  - 重置链接只能使用一次。
  - 修改密码后，自动登出所有设备。

## 3. 非功能需求
- **安全性**：密码必须加密存储（如 bcrypt），传输过程使用 HTTPS。
- **性能**：登录响应时间不超过 500ms。
- **兼容性**：支持主流浏览器（Chrome, Firefox, Safari, Edge）及移动端适配。
